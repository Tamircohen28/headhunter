# Changelog

The canonical changelog lives at the repository root: [../CHANGELOG.md](../CHANGELOG.md).

It follows [Keep a Changelog](https://keepachangelog.com/) and [Semantic Versioning](https://semver.org/).

When you land a user-facing change, add an entry under the `## [Unreleased]` section of the
root changelog. Releases move those entries into a dated version section — see the
[versioning policy](engineering/build-and-release/versioning.md) for how and when.
