# Architecture Decision Records

This directory records significant architectural decisions made in HeadHunter. Each ADR documents what was decided, why, and what alternatives were considered.

## Format

ADRs use a lightweight format: context, decision, rationale, alternatives, and consequences.

## Index

| ADR | Title | Status |
|-----|-------|--------|
| [001](001-local-json-store.md) | Local flat-JSON store instead of a database | Accepted |

## Adding a new ADR

1. Copy the format from an existing ADR.
2. Name it `NNN-short-title.md` (next sequential number).
3. Add it to the index table above.
4. Reference it from the PR description.
